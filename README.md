# Bitrix24 Delivery Bot

Бот для группового чата Битрикс24: курьеры пишут `/готово Накладная №1234`,
бот сохраняет факт доставки и в 18:00 (Астана) присылает заведующему складом
сводный отчёт за день.

## Структура

- `main.py` — FastAPI-приложение, входящий вебхук, планировщик
- `bitrix.py` — клиент REST API Битрикс24
- `database.py` — SQLAlchemy-модели (SQLite)
- `report.py` — формирование и отправка отчёта
- `config.py` — переменные окружения
- `render.yaml` — манифест для render.com
- `requirements.txt` — зависимости

## Локальный запуск

```bash
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # заполнить значения
uvicorn main:app --reload --port 8000
```

## 1. Настройка входящего вебхука (Битрикс24 → бот)

В Битрикс24 «входящий вебхук» от внешнего сервиса называется **исходящим вебхуком**
(события, которые портал шлёт наружу).

1. Зайдите в портал Битрикс24 как админ.
2. Левое меню → **Разработчикам** → **Другое** → **Исходящий вебхук**.
3. Параметры:
   - **URL вашего обработчика**: `https://<имя-сервиса>.onrender.com/bitrix/webhook`
   - **Код приложения / Токен (application_token)**: вставьте сюда то же значение, которое положите в переменную `SECRET_TOKEN`.
   - **Событие**: `ONIMMESSAGEADD` (новое сообщение в чате/мессенджере).
4. Сохранить.

> Альтернатива: можно зарегистрировать чат-бота через `imbot.register` и слушать
> `OnImBotMessageAdd` — но для приватной задачи проще использовать `ONIMMESSAGEADD`.

### Исходящий вебхук для отправки сообщений (бот → Битрикс24)

Это **второй** вебхук — обычный «Входящий вебхук» (в терминологии Битрикс24):

1. Левое меню → **Разработчикам** → **Другое** → **Входящий вебхук**.
2. Выдайте права: `im` (мессенджер), `user` (пользователи).
3. Скопируйте URL вида `https://your.bitrix24.ru/rest/1/abcdef123456/` —
   это значение переменной `BITRIX_WEBHOOK_URL`.

## 2. Деплой на render.com (пошагово)

1. Залейте проект в GitHub-репозиторий.
2. Зайдите на https://dashboard.render.com → **New** → **Blueprint**.
3. Подключите ваш репозиторий — Render автоматически прочитает `render.yaml`.
4. На шаге переменных окружения заполните:
   - `BITRIX_WEBHOOK_URL` — URL входящего вебхука Битрикс24 (с `/` на конце).
   - `BITRIX_CHAT_ID` — ID группового чата курьеров (см. п.3 ниже).
   - `MANAGER_USER_ID` — ID заведующего складом (см. п.3 ниже).
   - `SECRET_TOKEN` — длинная случайная строка; этот же токен укажите
     в исходящем вебхуке Битрикс24.
5. Нажмите **Apply** — Render склонирует репозиторий, поставит зависимости и
   поднимет сервис. Через 1–2 минуты будет доступен URL вида
   `https://bitrix-delivery-bot.onrender.com`.
6. Вернитесь в Битрикс24 и впишите этот URL + `/bitrix/webhook` в исходящий вебхук.

> ⚠️ На бесплатном плане Render сервис «засыпает» после 15 минут неактивности.
> Первый запрос после простоя обработается с задержкой ~30 сек. Для продакшена
> либо переходите на платный план, либо настройте пинг (например, UptimeRobot).
> SQLite-база живёт на временном диске и **обнуляется при каждом редеплое** —
> для долгого хранения подключите Render Disk или Postgres.

## 3. Как узнать BITRIX_CHAT_ID и MANAGER_USER_ID

### MANAGER_USER_ID

- Откройте профиль сотрудника в Битрикс24 — ID будет в URL: `/company/personal/user/7/` → ID = `7`.
- Либо через REST: откройте в браузере (подставив свой webhook URL)
  `https://your.bitrix24.ru/rest/1/TOKEN/user.search.json?FIND=Иванов` —
  в ответе будет поле `ID`.

### BITRIX_CHAT_ID

Самый простой способ — через REST-метод `im.recent.get`:

```
https://your.bitrix24.ru/rest/1/TOKEN/im.recent.get.json
```

В ответе массив `result.items`, у каждого чата поле `id` вида `"chat42"` — число после `chat` это и есть `BITRIX_CHAT_ID` (в нашем случае `42`).

Альтернатива — `im.chat.get` после того, как добавите бота в чат, или
открыть чат в веб-версии Битрикс24 и посмотреть URL.

## 4. Проверка работы после деплоя

1. **Healthcheck**: открыть в браузере `https://<имя>.onrender.com/` → должно вернуться `{"status":"ok"}`.
2. **Логи**: на render.com во вкладке **Logs** убедиться, что есть строка
   `Планировщик запущен — отчёт каждый день в 18:00 (Asia/Almaty)`.
3. **Тест вебхука**: написать в групповом чате `/готово Накладная №TEST-1`.
   В логах Render появится `Сохранена доставка: №TEST-1 — <ваше имя>`.
4. **Тест отсутствия номера**: написать просто `/готово` — бот ответит
   `⚠️ Пожалуйста, укажите номер накладной...`.
5. **Принудительно запустить отчёт**, не дожидаясь 18:00:
   ```
   curl -X POST "https://<имя>.onrender.com/admin/run-report?token=<SECRET_TOKEN>"
   ```
   В личке заведующего появится сегодняшний отчёт.

Если что-то не приходит — проверьте:
- `BITRIX_WEBHOOK_URL` заканчивается на `/`;
- у входящего вебхука Битрикс24 есть права `im` и `user`;
- `SECRET_TOKEN` в Битрикс24 и в env Render совпадают.
